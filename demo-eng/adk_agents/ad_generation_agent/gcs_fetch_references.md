# GCS Fetch References in `ad_generation_agent`

This document summarizes findings regarding how the `ad_generation_agent` fetches or references data from Google Cloud Storage (GCS).

## 📊 Summary of Mechanisms

The agent interacts with GCS for fetching data in two primary ways:
1.  **Direct GCS API Calls (via `adk_common.utils.utils_gcs`)**: Explicitly listing files or downloading text configurations.
2.  **Indirect Asset Loading (via `adk_common.utils.utils_agents.load_resource`)**: Dynamically loading binary assets (images, videos) for prompt engineering or model input based on URIs.

---

## 🔍 Detailed References

### 1. Direct GCS Fetching (Metadata & Configs)

These tools explicitly call GCS utilities to list or read files from buckets.

| File Path | Function | Description |
| :--- | :--- | :--- |
| `retrieve_generated_assets.py` | `retrieve_generated_assets` | Uses `get_files_metadata_from_gcs_bucket` to list all assets generated in a session or folder. |
| `retrieve_brand_identity.py` | `retrieve_brand_identity` | Uses `get_text_files_from_gcs_bucket` to load `.toml` brand blueprints. |

---

### 2. Indirect Asset Loading (Images & Videos)

All generation tools use `utils_agents.load_resource` to read assets. If the URL provided is a `gs://` URI, this function pulls the bytes down.

| File Path | Target Assets |
| :--- | :--- |
| `generate_ad_hoc_image.py` | Asset sheet, product image, logo, main character. |
| `generate_scene_frame.py` | References per scene frame. |
| `generate_asset_sheet.py` | Logo, product, main character. |
| `generate_video.py` | Reference images and base scene frames. |
| `combine_video.py` | Individual generated video clips. |
| `generate_display_ad.py` | Design references. |
| `evaluate_ad.py` | Newly generated asset vs reference URIs. |

---
