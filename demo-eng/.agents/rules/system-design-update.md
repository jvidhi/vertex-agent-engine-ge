When changes are done to the logic or when decisions are made (even if they don't require code changes, the SYSTEM_DESIGN.md file should be updated. The objective of that file is to keep a robust undertanding of the code, functional and non-functional requirements as well as all design decisions together with tradeoffs. This is meant to be an ever growing context of the project so that a new agent can read it and fully understand the code and the decisions thus far.
